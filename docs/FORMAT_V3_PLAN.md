# Format v3 Migration Plan

Format v2 is stable for the TypeScript LEVEL_0 MVP, but it has an unaligned uint32 field:

- rotationSeed at offset 21

JavaScript DataView supports this. Rust/Zig/C/WASM readers must parse v2 byte-by-byte.

Format v3 should realign fields:

| Offset | Size | Field |
|---:|---:|---|
| 0 | 4 | magic "TQMC" |
| 4 | 2 | version uint16 |
| 6 | 2 | flags uint16 |
| 8 | 4 | headerLength uint32 |
| 12 | 4 | dimensions uint32 |
| 16 | 4 | paddedDimensions uint32 |
| 20 | 4 | vectorCount uint32 |
| 24 | 1 | bitsPerValue uint8 |
| 25 | 1 | rotationMode uint8 |
| 26 | 1 | algorithmLevel uint8 |
| 27 | 1 | reserved0 uint8 |
| 28 | 4 | seed uint32 |
| 32 | 4 | normsOffset uint32 |
| 36 | 4 | normsLength uint32 |
| 40 | 4 | quantizedOffset uint32 |
| 44 | 4 | quantizedLength uint32 |
| 48 | 4 | qjlOffset uint32 |
| 52 | 4 | qjlLength uint32 |
| 56 | 4 | payloadCrc32 uint32 |
| 60 | 4 | headerCrc32 uint32 |
| 64 | 8 | reserved |

Migration rules:

1. v2 decoder must remain supported for all v3.x releases.
2. v3 encoder should be opt-in until WASM/Rust backend exists.
3. v3 must include test vectors generated by TypeScript and Python oracle.
4. v3 must reject unknown major versions.
